## 개념 1 — 인터넷 vs 웹: 헷갈리는 두 단어

| 용어 | 정의 | 비유 |
|------|------|------|
| **인터넷** | 전 세계 컴퓨터를 연결한 네트워크 그 자체 | 도로망 전체 |
| **웹(Web)** | 인터넷 위에서 동작하는 서비스 (HTTP 기반) | 도로 위를 달리는 자동차 |

**핵심:** 인터넷이 없으면 웹도 없지만, 인터넷에는 웹 말고도 이메일·FTP·SSH 등 다른 서비스도 있습니다.

---

## 개념 2 — IP 주소: 컴퓨터의 우편 주소

- 인터넷에 연결된 모든 컴퓨터는 고유한 IP 주소를 가짐
- **비유:** 집 주소처럼, IP 주소가 없으면 데이터를 어디로 보낼지 알 수 없음

```
내 컴퓨터: 192.168.1.10
네이버 서버: 125.209.222.141
```

**IPv4 vs IPv6:**

| 종류 | 형식 | 주소 개수 |
|------|------|----------|
| IPv4 | 192.168.0.1 | 약 43억 개 (부족) |
| IPv6 | 2001:db8::1 | 사실상 무한대 |

## 개념 3 — TCP: 데이터 분할 배송 시스템

인터넷으로 데이터를 보낼 때 한 번에 통째로 보내지 않습니다.

```
큰 파일 (10MB)
    ↓ 분할
패킷 1 / 패킷 2 / 패킷 3 / ... / 패킷 N
    ↓ 각자 다른 경로로 전송
목적지에서 순서대로 재조립
```

**TCP의 3가지 보장:**

| 기능 | 설명 | 비유 |
|------|------|------|
| 분할 전송 | 큰 데이터를 패킷으로 쪼개서 전송 | 택배 분할 포장 |
| 순서 보장 | 도착 순서가 달라도 올바르게 재조립 | 박스마다 번호 스티커 |
| 재전송 | 분실된 패킷 자동 재요청 | 배송 추적 + 미수령 시 재배송 |

---

## 개념 4 — HTTP: 웹의 주문서 양식

HTTP = HyperText Transfer Protocol — 웹 브라우저와 서버가 대화하는 규칙

**비유:** 식당의 표준 주문서 양식
- 손님(브라우저)이 정해진 형식으로 주문(요청)하면
- 주방(서버)이 정해진 형식으로 음식(응답)을 내어 줌

```
요청: GET /index.html HTTP/1.1
응답: HTTP/1.1 200 OK + HTML 내용
```

| 상태 코드 | 의미 |
|---------|------|
| 200 | 성공 |
| 404 | 페이지 없음 |
| 500 | 서버 오류 |
| 301 | 주소 이동됨 |

**HTTPS** = HTTP + 암호화 (자물쇠 아이콘 = HTTPS)

## 개념 5 — 라우터: 교차로 신호등

데이터 패킷이 목적지까지 가는 최적 경로를 결정하는 장비

- **비유:** 교차로의 신호등 + 내비게이션
  - "이 데이터는 저 방향으로 보내라" 경로 판단
  - 가정에서 쓰는 공유기(Wi-Fi 라우터)가 대표적

```
내 컴퓨터 → 가정용 라우터 → ISP 라우터 → 인터넷 → 목적지 서버
```

---

## 개념 6 — 스위치: 사무실 내선 교환기

같은 네트워크 안에서 컴퓨터끼리 연결해주는 장비

| 장비 | 역할 | 범위 |
|------|------|------|
| **스위치** | 같은 건물(LAN) 안 컴퓨터 연결 | 사무실 내부 |
| **라우터** | 다른 네트워크(WAN)와 연결 | 인터넷 구간 |

**비유:** 스위치 = 사무실 내선 전화 교환기, 라우터 = 외부 전화 연결 교환기

---

## 개념 7 — 방화벽: 건물 보안 게이트

허용된 트래픽만 통과시키고 나머지는 차단하는 보안 장비

**비유:** 건물 입구 보안 게이트
- 신분증(허용된 포트) 있으면 입장
- 신분증 없으면 차단

**주요 허용 포트:**

| 포트 | 용도 | 허용 여부 |
|------|------|----------|
| 443 | HTTPS (보안 웹) | 허용 |
| 80 | HTTP (일반 웹) | 보통 허용 |
| 22 | SSH (서버 관리) | 관리자만 허용 |
| 3306 | MySQL DB | 내부망만 허용 |

## 개념 8 — CDN: 전국 편의점 물류 거점

Content Delivery Network — 전 세계에 콘텐츠 복사본을 배치해 빠르게 전달하는 시스템

**비유:** 전국 편의점 물류 거점
- 서울 본사 창고(원본 서버)에서 전국 편의점(CDN 서버)에 미리 배포
- 부산 고객은 부산 편의점에서 바로 받음 → 빠름

```
미국 서버에 접속 (지연 200ms)
         vs
한국 CDN 서버에 접속 (지연 10ms) ← CDN 효과
```

**대표 서비스:** Cloudflare, AWS CloudFront, Akamai

## 개념 9 — DMZ: 외부 손님 대기실

De-Militarized Zone — 외부 인터넷과 내부망 사이의 중간 구역

**비유:** 회사 로비(대기실)
- 외부 손님은 로비까지만 입장 가능
- 내부 직원 구역(내부망)은 로비를 통해서만 접근

```
인터넷
   ↓
[방화벽]
   ↓
 DMZ (웹 서버, 메일 서버 등 외부 노출 서버)
   ↓
[방화벽]
   ↓
 내부망 (WAS, DB — 외부 직접 접근 불가)
```

## 개념 10 — Web/WAS/DB 이중화 아키텍처: 실제 서비스 구성

실제 운영 서비스의 표준 3계층 구조

```
사용자
  ↓ HTTPS (443)
[로드밸런서]
  ↓
[Web 서버] ← 정적 파일(HTML, 이미지) 처리
  ↓ HTTP (8080)
[WAS 서버] ← 비즈니스 로직, 동적 처리 (Spring, Django 등)
  ↓ TCP (3306)
[DB 서버]  ← 데이터 저장
```

**이중화 (High Availability):**

| 계층 | 이중화 방법 | 이유 |
|------|-----------|------|
| Web/WAS | 동일 서버 2대 + 로드밸런서 | 1대 다운 시 나머지로 서비스 |
| DB | Primary(쓰기) + Standby(읽기/복제) | 데이터 손실 방지 |

**비유:** 주방 2개(Web/WAS 이중화) + 금고 이중 잠금(DB Primary/Standby)

## 핵심 10개 용어 한눈에 정리

| 용어 | 한 줄 설명 |
|------|-----------|
| 인터넷 | 전 세계 컴퓨터 연결 도로망 |
| 웹 | 인터넷 위에서 달리는 자동차 (HTTP 기반 서비스) |
| IP 주소 | 컴퓨터의 우편 주소 |
| TCP | 데이터 분할 배송 + 순서 보장 + 재전송 시스템 |
| HTTP | 브라우저-서버 간 주문서 양식 규칙 |
| 라우터 | 데이터 경로를 결정하는 교차로 신호등 |
| 스위치 | 같은 네트워크 안 컴퓨터 연결 교환기 |
| 방화벽 | 허용 포트만 통과시키는 보안 게이트 |
| CDN | 전국 편의점 물류 거점 (콘텐츠 분산 배포) |
| DMZ | 외부-내부망 사이 손님 대기실 |
| Web/WAS/DB | 실제 서비스의 3계층 표준 아키텍처 |

## 네트워크 기초 실전 팁 3가지

1. **포트 번호는 외워두기** — 443(HTTPS), 80(HTTP), 22(SSH), 3306(MySQL), 8080(WAS) 5개만 알면 대부분의 대화 가능
2. **오류 코드 200/404/500** — 200=정상, 404=주소 없음, 500=서버 문제. 이 세 가지로 문제 원인 1차 판단
3. **DMZ 개념이 핵심** — 보안 설계 논의 시 "DMZ에 두냐 내부망에 두냐"가 가장 자주 나오는 질문
