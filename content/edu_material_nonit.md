# 바이브 코딩을 위한 필수 IT 지식 10가지
## 비전공자 교육 자료 v1.0

> 대상: IT 비전공 직장인, AI 코딩 도구를 처음 접하는 분
> *IT 기초 개념을 비전공자 눈높이로 재구성한 교육 자료입니다.*

---

## 시작하기 전에

코딩을 배운 적 없지만 **AI 도구(ChatGPT, Claude)로 앱을 만들고 싶은 분**을 위한 자료입니다.
"바이브 코딩"이란 코드를 직접 짜지 않고 AI에게 말로 설명해서 프로그램을 만드는 방식입니다.
이 자료의 핵심 용어 10개만 이해하면 바이브 코딩을 시작할 수 있습니다.

## 개념 1 — AI 도구 선택: ChatGPT vs Claude vs Cursor

| 도구 | 비유 | 잘하는 것 |
|------|------|----------|
| ChatGPT | 똑똑한 비서 | 글쓰기, 요약, 질문 답변 |
| Claude | 개발 전문 비서 | 코드 작성, 긴 문서 분석 |
| Cursor / Claude Code | 개발 전용 작업실 | 파일 직접 수정, 프로젝트 전체 관리 |

**핵심:** 단순 질문은 ChatGPT, 실제 프로그램 만들기는 Claude Code나 Cursor를 쓰세요.

---

## 개념 2 — CLI vs IDE: 두 가지 작업 환경

**CLI (명령줄 인터페이스)**
- 비유: 주방에서 요리사에게 직접 말로 주문하는 것
- 생김새: 까만 화면에 글자만 있음 (Terminal, 명령 프롬프트)
- 사용 예: `npm start` → "서버 시작해!" 라고 타이핑

**IDE (통합 개발 환경)**
- 비유: 모든 도구가 갖춰진 주방 (칼, 도마, 레인지 한 곳에)
- 생김새: VS Code, Cursor — 파일 목록 + 코드 편집기 + 터미널 한 화면
- 바이브 코딩 시 주로 이 환경에서 작업

## 개념 3 — 소스코드 → 빌드 → 배포 흐름

```
내가 작성 (or AI가 작성)
       ↓
  소스코드 (.js .py .html 파일들)
       ↓
    빌드 (컴퓨터가 이해하는 형태로 변환)
       ↓
   배포 (인터넷에 올리기 → 전 세계 접속 가능)
```

**비유:** 레시피(소스코드) → 요리(빌드) → 식당에 내놓기(배포)

---

## 개념 4 — Git: 작업 내역 저장소

- **비유:** 문서의 "버전 관리" 기능 (Word의 "변경 내용 추적"의 강력한 버전)
- **핵심 동작 3가지:**
  - `commit` = "이 시점 저장해둬" 스냅샷 찍기
  - `push` = 내 컴퓨터 → 인터넷 저장소(GitHub)로 업로드
  - `pull` = 인터넷 저장소 → 내 컴퓨터로 다운로드
- **왜 중요한가:** AI가 코드를 잘못 수정했을 때 되돌리는 안전망

---

## 개념 5 — 프론트엔드 vs 백엔드

```
사용자 화면 (프론트엔드)          뒤에서 처리 (백엔드)
━━━━━━━━━━━━━━━━━━           ━━━━━━━━━━━━━━━━━━
버튼, 글자, 이미지              데이터 저장, 계산, 인증
HTML / CSS / JavaScript        Python / Node.js / Java
브라우저에서 실행               서버에서 실행
```

**비유:** 카페의 홀(프론트엔드) vs 주방(백엔드)
고객은 홀만 보지만, 실제 커피는 주방에서 나옴

## 개념 6 — 포트: 같은 컴퓨터 안의 방 번호

- 컴퓨터 하나에 여러 프로그램이 동시에 실행
- 포트 = 각 프로그램의 "방 번호"

| 포트 번호 | 용도 |
|----------|------|
| 80 | 일반 웹사이트 (http) |
| 443 | 보안 웹사이트 (https) |
| 3000 | 개발 중인 내 앱 테스트 |
| 3306 | MySQL 데이터베이스 |

**localhost:3000** = "내 컴퓨터의 3000번 방에서 실행 중인 앱"

---

## 개념 7 — API: 서비스끼리 대화하는 방법

**비유:** 식당의 메뉴판 + 주문 시스템
- 손님(앱A)이 메뉴(API 명세서)를 보고 주문(요청)을 하면
- 주방(앱B)이 요리(데이터)를 돌려줌

**4가지 주문 방식 (HTTP 메서드):**

| 방식 | 의미 | 예시 |
|------|------|------|
| GET | 가져와 | "사용자 목록 보여줘" |
| POST | 만들어 | "새 회원 등록해" |
| PUT | 수정해 | "이름 바꿔줘" |
| DELETE | 지워 | "계정 삭제해" |

---

## 개념 8 — JSON: 데이터를 주고받는 형식

- **비유:** 표준화된 배송 박스 (어떤 택배사든 같은 규격)
- 사람도 읽을 수 있는 텍스트 형태

```json
{
  "이름": "홍길동",
  "나이": 42,
  "직업": "공무원"
}
```

API가 데이터를 주고받을 때 이 형식을 사용

## 개념 9 — 데이터베이스: 앱의 저장 창고

| 종류 | 비유 | 대표 도구 | 특징 |
|------|------|----------|------|
| SQL (관계형) | 엑셀 스프레드시트 | MySQL, PostgreSQL | 행·열 구조, 정형 데이터 |
| NoSQL | 자유형 서랍장 | MongoDB | 자유로운 형식, 빠른 개발 |

**바이브 코딩 팁:** AI에게 "MySQL 써줘" 또는 "MongoDB 써줘" 중 하나만 지정하면 됨

## 개념 10 — 배포 서비스: 인터넷에 올리기

| 서비스 | 특징 | 추천 대상 |
|--------|------|----------|
| Vercel | 프론트엔드 최강, 무료 시작 | 웹사이트 첫 배포 |
| Cloudflare | 전 세계 빠른 전송 | 트래픽 많을 때 |
| AWS | 기업급, 복잡하지만 강력 | 대규모 서비스 |

**초보자 추천:** Vercel → GitHub에 push하면 자동으로 인터넷에 올라감

## 바이브 코딩 실전 팁 3가지

1. **비유로 설명하기** — AI에게 "엑셀처럼 데이터 저장해줘"처럼 비유로 요청하면 더 잘 이해함
2. **에러는 그대로 복사** — 오류 메시지 발생 시 전체를 복사해서 AI에게 붙여넣기
3. **한 번에 하나씩** — "로그인 기능만 먼저 만들어줘" → 동작 확인 → 다음 기능 순서로 진행
